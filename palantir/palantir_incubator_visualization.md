# Palantir 终极创始人孵化器 - 可视化交互网页 PRD

## 1. 项目背景
基于文章《Palantir 如何打造终极创始人孵化器：与 Nabeel S. Qureshi 的深度对话》，构建一个可视化的动态交互网页。
**目标**：通过角色化视角和可视化交互，帮助不同岗位的读者（部门负责人、产品经理、架构师、研发人员）快速获取感兴趣的核心信息，大幅缩减阅读时间。

## 2. 核心设计理念
- **以人为本 (Persona-First)**：不再按文章线性结构展示，而是按“用户角色”重组信息。
- **动态交互 (Dynamic Interaction)**：用户选择角色后，页面内容、高亮区域、数据图表实时变化。
- **可视化 (Visualization)**：使用知识图谱、时间轴、卡片墙代替纯文本。

## 3. 目标用户与需求分析

| 用户角色 | 核心关注点 | 期望获取的信息 |
| :--- | :--- | :--- |
| **部门负责人 (Dept Head)** | 人才策略、组织文化、商业模式 | 如何筛选顶级人才（反向筛选）、去职级化管理优势、从咨询到产品的转型（利润率提升）、FDE 模式的投入产出比。 |
| **产品经理 (PM)** | 产品演进、用户洞察、迭代流程 | 定制化服务如何沉淀为通用平台（Foundry/Gotham）、Ontology（本体）设计的用户同理心、PM 与 FDE 的关系、快速迭代验证（周更）。 |
| **架构师 (Architect)** | 技术架构、数据模型、系统集成 | 平台三层架构（采集/处理/交互）、Gotham 与 Foundry 的技术差异（图分析 vs 表格处理）、Ontology 数据建模解决“数据孤岛”的方法、权限管理（Gatekeepers）。 |
| **研发人员 (Developer)** | 工作模式、技术栈、职业成长 | FDE 的实战日常（驻场开发 vs 远程）、全栈技术要求、解决实际问题的成就感、如何利用 AI 工具（Whisper, Claude）、Palantir 如何成为创业学校。 |

## 4. 功能模块设计

### 4.1 核心功能：角色透镜 (Role Lens)
- **交互**：页面顶部设有显眼的“角色切换器”（Tabs 或 浮动导航）。
- **效果**：
    - 切换到 **“部门负责人”**：高亮展示“招聘哲学”、“组织架构”、“商业转型”模块；隐藏技术细节。
    - 切换到 **“架构师”**：高亮“数据平台架构”、“Ontology 模型”、“技术栈”；折叠管理类内容。
    - 提供 **“全览模式”**：展示所有内容。

### 4.2 模块一：Palantir 核心飞轮 (可视化图谱)
- 一个动态的循环图或网状图，连接以下节点，点击节点展开详情：
    - **FDE (一线部署工程师)**：连接“客户痛点”与“产品研发”。
    - **反向筛选**：连接“顶级人才”与“公司使命”。
    - **Ontology (本体)**：连接“数据孤岛”与“业务决策”。
    - **利润率**：连接“定制服务”与“标准化平台”。

### 4.3 模块二：时间轴演进 (The Evolution)
- 横向滚动时间轴，展示从“咨询公司”到“软件巨头”的关键转折点：
    - **早期**：Jupyter Notebooks + 驻场 = 解决单点问题。
    - **中期**：提炼通用工具 -> 强制使用内部工具 (Dogfooding)。
    - **成熟期**：Foundry/Gotham 平台化 -> 80% 利润率。

### 4.4 模块三：深度问答 (Ask the Expert)
- 预设关键问题（根据角色变化）：
    - (PM) "如何平衡定制需求与通用产品？"
    - (Leader) "为什么不设职级？"
    - (Dev) "FDE 是售前还是开发？"
- 点击问题显示 Nabeel 的精炼回答（引用原文）。

## 5. 页面结构与 UI 建议

### 5.1 页面布局
1.  **Hero Section**：酷炫的 dark mode 背景，标题“Palantir: The Founder Factory”，副标题“选择你的角色，解锁你的成长路径”。
2.  **Role Selector**：四个大卡片/图标，带有微动效。
3.  **Main Content Area**：
    - 左侧：动态导航/大纲。
    - 中部：卡片式内容流（根据角色过滤）。
    - 右侧（或悬浮）：关键概念解释（Ontology, FDE, Reverse Filtering）。

### 5.2 视觉风格
- **色调**：Palantir 风格的深色系（黑、深灰），搭配霓虹蓝/紫作为强调色。
- **排版**：高对比度，关键数字（如 80%、30%）放大显示。
- **动效**：元素入场淡入淡出，角色切换时内容块的重新排列动画（Masonry layout transition）。

## 6. 技术实现建议
- **架构**: 单文件 HTML (Single File HTML)。
- **样式**: Tailwind CSS (通过 CDN 引入)。
- **交互**: Vanilla JavaScript (原生 JS) 或轻量级库 (如 Alpine.js)。
- **图表**: Chart.js 或 ECharts (通过 CDN 引入，如果需要复杂图表)。
- **部署**: 直接打开文件即可，也可部署到 GitHub Pages。
